Browser Automation Rules

The Golden Navigation System

Non-negotiable rules for all browser automation pipelines. Violation of any rule is a pipeline failure.

Core Philosophy: "Look Before You Leap" (LB4YL)

Scan the page, discover all selectors, screenshot to verify — then act. Never interact with a page element you haven't first discovered through the registry.

Rule 5 — Auth & Sessions

Persistent Profiles via OpenClaw

🗂️ Profile Storage

  • Profiles live at ~/.openclaw/<platform>-chrome-profile
  • Uses chromium.launchPersistentContext(profileDir)
  • Cookies, sessions, auth tokens persist across runs
  • No re-login on next script execution

🚫 Never Ephemeral

  • Never launch a fresh browser for authenticated flows
  • Closing a persistent profile destroys session state
  • For multi-step OAuth/OTP: also save cookies to tools/_cookies/{domain}.json
  • Reuse browser across multiple platforms in the same pipeline run

🔒 Browser Close Rule

  • browser.close() is the absolute last line
  • Only called after ALL objectives verified with screenshots
  • On failure mid-flow: stay in browser and debug
  • When in doubt — leave it open
Rule 3 — Discovery Engine

SelectorRegistry — How Discovery Works

1

Cache Check

If a previously-verified selector is still visible, reuse it immediately. Zero network cost.

2

Candidate List

Try provided candidate selectors in order — first visible candidate wins.

3

Screenshot the Winner

Screenshot the winning element → save to registry JSON with verified_at and url_context.

4

Semantic DOM Fallback

All candidates failed? Run DOM_SCANNER on the full page — score every visible element by keyword matching against semanticHint. Build a precise selector from the best match.

5

Failure Dump

Total failure? Write full DOM JSON to ~/.openclaw/selector-registry/failures/ for forensic inspection.


Persistent selector dictionary stored at ~/.openclaw/selector-registry/<site>.json — reused by all future scripts on that site.

// Import once per script
const SelectorRegistry = require('./selector-registry');
const reg = new SelectorRegistry('gumroad'); // one instance per site

// ✅ RIGHT — all selectors through the registry
const sel = await reg.discover(page, 'email_field', {
  candidates: ['input[name="email"]', 'input[type="email"]'],
  semanticHint: 'email login username',
});

// ❌ WRONG — never write bare locators inline
// page.locator('input[name="email"]')
Rule 4 — Proof Trail

Triple Verification on Every Action

// Triple verification — every fill follows this pattern
const el = await page.locator(sel);

// 1. Screenshot BEFORE
await ssEl(page, sel, 'headline_BEFORE');

// 2. Fill value
await el.fill(targetValue);

// 3. Screenshot AFTER
await ssEl(page, sel, 'headline_AFTER');

// 4. Verify value stuck — programmatic check
const got = await el.inputValue();
if (got.trim() !== targetValue.trim()) {
  reg.strike('headline', 'fill failed — value did not stick');
}
Complete Reference

All 11 Rules

Rule 0

Golden Sequence — SCAN → DISCOVER → SS → ACT → SS → VERIFY

Every single interaction follows this sequence. On any page state change, restart from SCAN.

Rule 1

Never Close the Browser Until Fully Verified

browser.close() is the absolute last line — only after all objectives are verified with screenshots. A lingering browser costs nothing; a destroyed session costs everything.

Rule 2

No Hard Reloads

Never use page.reload(). Use waitForSelector() or waitForLoadState("networkidle"). Add human-paced waits (800–1800ms) between actions.

Rule 3

SelectorRegistry for All Selectors — No Exceptions

Every selector — including file inputs, buttons, labels, hidden inputs — goes through reg.discover(). Never write bare page.locator() inline.

Rule 4

Screenshot at Every Decision Point

Before action, after action, and after verification. Screenshots are the proof trail. Triple verification on every fill and upload.

Rule 5

Persist Browser State — Persistent Profiles Only

Use chromium.launchPersistentContext(profileDir). Profiles at ~/.openclaw/<platform>-chrome-profile. Never launch ephemeral browsers for authenticated flows.

Rule 6

3-Strike Rule — Stop and Report After 3 Failures

After 3 failures on any field: stop the sub-step entirely. Generate a failure report with DOM dump, screenshots, tried selectors, and error messages. Never retry blindly.

Rule 7

Re-scan After Every Page State Change

After any click, navigation, upload, save, or scroll that might change the DOM — re-scan before the next action. This rule is the most commonly violated.

Rule 8

Background Browsers for Multi-Step Flows

For flows requiring human-like delays (OTP emails, OAuth redirects), launch the browser as a background process. Prevents timeout kills from destroying the session.

Rule 9

Fill All Available Fields — 100% Completeness

Scan the page to discover every field and image upload slot. If a field exists and data is available, it must be filled. Don't skip elements because they're hard.

Rule 10

CAPTCHA — Always Solve Automatically via Audio

Never wait for manual solve. Always switch to audio challenge mode. Use the shared captcha-solver.js — download MP3 → convert → transcribe via Windows Speech → submit. Never write inline CAPTCHA handlers.

Origin

Real Failures — Why These Rules Exist

Hardcoded Selectors

Inline selectors like input[accept=".jpeg,.jpg,.png"] break when the site changes its DOM structure.

No Re-scan After Click

Clicked a button that revealed a file input, then tried to discover the new element from stale DOM state.

Missing Upload Slots

Didn't scan for all image upload areas — missed that a platform had a separate logo/icon slot.

No Verification After Upload

Uploaded a file but didn't screenshot or re-scan to confirm the preview image actually appeared.

Browser Closed Per-Platform

Lost auth state by closing the browser after each platform — forced re-login on every run.

Manual CAPTCHA Waiting

Scripts waited 60–180s hoping the user would solve it, or just clicked the checkbox and hoped for the best.

Browser Closed Before Verification

Job appeared complete but changes hadn't persisted. No screenshot proof. No way to confirm.

Inline CAPTCHA Selectors

Hardcoded iframe[title*="reCAPTCHA"] instead of routing through the shared captcha-solver.js module.

These are now rules precisely because they were violated. Every rule is a documented scar.
